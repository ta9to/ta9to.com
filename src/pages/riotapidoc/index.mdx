# RIOT DEVELOPER PORTAL 日本語
※<a target="_blank" href="https://developer.riotgames.com">Riot Developer Portal</a>の非公式日本語翻訳ドキュメントです。

## 概要 {{ id: 'overview' }}

このドキュメントは、Riot Games 開発者ポータルについての基本的な理解を提供します。これは、Riotのゲームのプレイヤーのための経験にプラスの影響を与えるために、独自のツールや製品の探求と開発を開始するのに役立つように設計されています。私たちは、あなたが何を作るか見るのを待つことができません!

## はじめに {{ id: 'getting-started' }}

開発者ポータルの利用を開始する前に、Riot Gamesのアカウントで<a target="_blank" href="https://auth.riotgames.com/login#client_id=riot-developer-portal&redirect_uri=https%3A%2F%2Fdeveloper.riotgames.com%2Foauth2-callback&response_type=code&scope=openid%20email%20summoner&ui_locales=en">ログイン</a>する必要があります。ログインすると、Developer Portal のアカウントが作成されます。この操作で、あなたのアカウントに関連する基本的な開発APIキーも生成されます。このドキュメントでは、このキーを強化するためにRiotとどのように対話できるかを説明します。

このアカウントは、RiotのDeveloper Relationsチームにあなたの製品案を<a target="_blank" href="https://developer.riotgames.com/app-type">登録</a>するオプションを提供します。一度製品を登録すると、承認やアクセス権の増加など、私たちのチームとコミュニケーションをとることができます。

# 製品登録 {{ id: 'product-registration' }}

ここにいらっしゃる方は、Riotのゲームのためにクールなアプリケーションやウェブサイトを作ろうと思っているのではないでしょうか。わかりやすくするために、そういったものを大まかに「プロダクト」と呼ぶことにします。前述の通り、Developer Portalにログインすると、プロダクトの企画書を登録することができるようになります。その内容を説明しましょう。

## 応募方法 {{ id: 'application-process' }}

まず最初に、すべてのプロダクトオーナー（開発者）は、私たちのポリシーを読み、理解することが期待されます。このページの上部にあるナビバーで見ることができます。必ずお読みください。これらのポリシーに従わない場合、懲罰的な措置が取られることがありますので、あまり前に進む前に理解しておくようにしてください。

あなたの製品がRiot Games APIを使用しない場合でも、製品を登録することが有益であることがわかります。登録することで、あなたがそこで素晴らしいことをしていることをRiotに知らせることができます。登録することで、Riotが貴社を知り、貴社製品に有益な情報を提供することができます。また、Riotがあなたや他の開発者をどのように支援できるかを私たちに知らせる道も開けます。

では、どうすれば登録できるのでしょうか？

それは簡単です。開発者ポータルのメインページに戻り、「製品登録」ボタンをクリックします。そこで、大規模な製品なのか、個人的なプロジェクトなのかを決定する必要があります。次に、製品に関する重要な情報を収集するためのオンライン・フォームに入力していただきます。最後に、本当に開発者であることを確認するために、製品の検証をお願いします。

これらのステップを完了すると、デベロッパーリレーションズチームがあなたの提案を検討します。そして、APIキーの追加レート制限を解除し、開発チームと関係を築きながら、製品の改良に取り組んでいただきます。

製品が承認された場合は、適用されるすべての規則、ポリシー、法律に準拠して製品を維持することが期待されます。それらのすべてについて最新の情報を入手することは、あなたの責任です。そのために助けが必要な場合は、開発者ポータルにメッセージを残すか、<a target="_blank" href="https://discord.com/invite/riotgamesdevrel">開発者ディスコード</a>に飛び込んでください。

もしあなたの製品提案が却下された場合は、必ずデベロッパーポータルにメッセージを残します。私たちはあなたの成功を願っています。私たちの懸念に対処し、あなたの素晴らしいアイデアをプレイヤーの手に届けるために、メッセージシステムを通じて私たちと自由に連携してください。

アプリケーションの審査では、それがRiot Gamesのエコシステムに利益をもたらすものであるかどうかを判断するために、多くの要素に注目します。特に、その製品が法律や既存のポリシーに違反していないことを確認したいと思います。また、その製品が何らかの測定可能な方法でプレイヤーの役に立つかどうかも確認したいと思います。私たちは、プレイヤーのゲーム上達を支援する製品や、プレイヤーの成長を追跡する製品を望んでいます。しかし、ゲームを解決するような製品や、すべてを単純化するような製品は見たくありません。また、私たちはクオリティも重視しています。ウェブサイトが完全でない場合、製品を承認する可能性は低くなります。もし質問がある場合は、デベロッパーポータルのメッセージシステムを通じてご連絡ください。

## メッセージ {{ id: 'messages' }}

この時点ですでに開発者ポータルにログインしているため、「メッセージ」を送信することも可能です。

製品登録に関連する質問がある場合は、メッセージを送信してフィードバックを得てください。メッセージは、デベロッパーリレーションズチームから回答を得るための最も確実で迅速な方法です。さらに、あなたのライブ製品やエコシステムの他の何かについて質問がある場合は、メッセージを投稿していただければ、私たちが対応します。

注意：メッセージシステムを無責任に使用しないでください。これは、あなたがRiot社員と気軽にチャットするためのチャンネルではありません。これはビジネスチャンネルであり、そのように扱われるべきです。

# Web APIs {{ id: 'web-apis' }}

あなたの製品は、Riot Games APIに依存している可能性があります。その場合、Riot Games APIキーに関する基本的な概念を理解していることを確認する必要があります。

## API Keys {{ id: 'api-keys' }}

開発者ポータルにサインインした人には、そのアカウントに関連付けられたAPIキーが自動的に付与されます。このAPIキーにより、お客様はすぐにAPIの利用を開始することができ、お客様のキーに対応したAPI利用状況のメトリックスを生成することができます。弊社では、全部で数種類のAPIキーを管理しています。以下の違いをご覧ください。

### 開発用API Key {{ id: 'development-api-key' }}

開発者ポータルにサインインした際に生成されたAPIキーは、**開発用APIキー**です。これらの暫定的なAPIキーは、一般消費者向けではないが、APIへの一時的なアクセスで利益を得る製品のために一時的に付与されます。開発用APIキーの目的は、Riot GamesのAPIをいじり、プロトタイプを開発し、コミュニティで利用できるようにすることです。また、このキーは24時間ごとに無効化されます。定期的にリセットする必要があります。

### 個人用API Key {{ id: 'personal-api-kye' }}

個人用APIキーは、製品を登録することで申請することができます。**個人用APIキー**は、開発者だけ、または小規模なプライベート・コミュニティを対象とした製品に使用する必要があります。これらの製品は、検証プロセスを経ずに登録することができますが、レート制限の引き上げは承認されません。Standard APIへのアクセスを要求することはできますが、Tournaments APIへのアクセスは要求できません。個人用キーには、製品の詳細な説明が必要です。

個人用APIキーの許容される用途は以下の通りです。

- ストリーミングサイト、掲示板、voice comサーバーなどのボット。
- 個人サイトの統計情報を表示する。
- 個人的な統計情報を収集するためのプロジェクト
- 個人的な研究
- 個人的な利用を目的とし、制作を目的としないプロジェクト

個人用鍵のレート制限は、設計上、非常に限られています。

- 20リクエスト/1秒
- 100 リクエスト/2 分

レート制限は地域ごとに実施されることに注意してください。例えば、上記のレート制限では、NAとEUWのLeague of Legendsエンドポイントに同時に1秒間に20回のリクエストをすることができます。

プロダクションキーの承認プロセスにかかる時間にかかわらず、個人キーを使用してアプリケーションを一般消費者向けに実行することはできません。一般消費には、オープンアルファ/ベータテストが含まれることに注意してください。

### プロダクションAPI Key {{ id: 'production-api-key' }}

Riot Games APIを使用したアプリケーションを開発し、プレイヤーに公開したい場合は、製品用APIキーを申請してください。開発用APIキーで公開製品を維持することはできません。**本番用APIキー**は、公開製品のトラフィックを維持するのに適した、より高いレート制限を備えています。

本番用APIキーは、大規模なコミュニティまたはインターネット全体を対象とする製品に使用する必要があります。Standard APIとTournaments APIへのアクセスを要求することができます。通常、APIキーを受け取る前に、動作するプロトタイプを必要とします。

本番用APIキーの開始レート制限は、開発用キーよりもはるかに大きくなっています。

- 10秒に500リクエスト
- 30,000 リクエスト/10 分
d
このレート制限は地域ごとに適用されることに注意してください。

レート制限を拡大した本番用キーを申請するには、ダッシュボードで「プロジェクトの登録」をクリックします。承認されたProduction Keyを取得するためのプロセスと所要時間は、プロジェクトとアプリケーションの対象地域によって異なる場合があります。

標準的な制作料金の上限は、大多数の開発者のニーズを満たしますが、開発者の状態が良好で、コミュニティへの貢献度が高く、標準的な制作料金の上限を着実に超過している場合は、上限を拡大することができることにご留意ください。

複数のプロジェクトに取り組んでいる場合は、それぞれを別々に登録し、それぞれが個別のプロダクション API キーを承認する必要があります。

### API Keyセキュリティ {{ id: 'api-key-security' }}

APIキーに関する最後の注意点として、キーが適切に保護されていない場合、キーが失効してしまう可能性が高いということを述べておきます。APIキーの保護は、<a target="_blank" href="https://developer.riotgames.com/policies/general">General Policies</a>に記載されているように、プロジェクトを公開するための必須条件です。もしあなたの鍵が適切に保護されていないと判断した場合、私たちはあなたのために鍵を保護する措置を取ります :)

## レスポンスコード {{ id: 'response-codes' }}

Riot Games APIは、すべてのデータを有効なJSONで返します。いくつかのプログラミング言語は、JSONのネイティブサポートを含んでいます。そうでないものについては、http://www.json.org で適切なライブラリを見つけることができます。

我々のAPIは、帯域幅を節約するために、空でない値のみを返すことに注意してください。ゼロは、空文字列、空リスト、ヌルと同様に、空値とみなされます。返されない数値フィールドはすべて0(あるいはNULL)と見なします。返されないリストフィールドは、空リストかヌルと見なすことができます。返されない文字列フィールドは、空文字列またはNULLとみなすことができます。

### 2xx レスポンスコード {{ id: '2XX-response-code' }}

200レスポンスコードについては、APIリファレンスページに記載されているレスポンスボディを常に期待することができます。200レスポンスコードのみ、レスポンスボディがJSONで返されることが保証されています。

200以外のレスポンスコードについては、以下の点にご注意ください。

レスポンスボディが返されることは保証されません。

レスポンスボディがある場合、その内容はJSONであることを保証していません。
現在、人間が読めるデバッグ情報をJSONで返していますが、このデバッグ情報の構成と内容は変更される可能性があります。例として...

```json
{
    "status": {
        "message": "Unauthorized",
        "status_code": 401,
    }
}
```

status、message、status_code の内容は、与えられた応答コードに対して常に存在し、一定であることは保証されない。4. 4. アプリケーション内のロジックは、レスポンスコードのみに基づいて優雅に失敗すべきであり、レスポンスボディに依存してはいけません。

### 4xx エラーコード {{ id: '4xx-error-code' }}

4xxクラスのエラーコードは、クライアントが有効なリクエストを提供できなかったことを示すためのものです。以下は、APIを使用する際に遭遇する可能性のある、最も一般的な4xxクラスのエラーコードです。

**400 Bad Request**このエラーは、リクエストに構文エラーがあり、そのためリクエストが拒否されたことを示します。クライアントは、構文やリクエストの内容を変更せずに、同様のリクエストを続けるべきではありません。

よくある理由

- 提供されたパラメータの形式が正しくない (例: 整数ではなく文字列)。
- 提供されたパラメータが無効です（たとえば、beginTime と startTime は大きすぎる時間範囲を指定しています）。
- 必須パラメーターが提供されていない。

**401 Unauthorized**このエラーは、作成されたリクエストに必要な認証情報 (例: API キー) が含まれていないため、クライアントがアクセスを拒否されたことを示します。クライアントは、リクエストにAPIキーを含めずに、同様のリクエストを続けるべきではありません。

よくある理由

- APIキーがリクエストに含まれていない。

**403 Forbidden**このエラーは、サーバーがリクエストを理解したものの、認可を拒否したことを示します。無効なパスと無効な認証情報 (例: API キー) は区別されません。クライアントは同様のリクエストを続けるべきではありません。

よくある理由

- APIリクエストに無効なAPIキーが指定された。
- ブラックリストに登録されたAPIキーがAPIリクエストで提供された。
- APIリクエストのパスが不正確、またはサポートされていないパスであった。

**404 Not Found**このエラーは、サーバーが、実行中のAPIリクエストに一致するものを見つけられなかったことを示します。この状態が一時的なものか恒久的なものかは、表示されません。

一般的な理由

- 指定されたIDまたは名前が既存のリソースと一致しない（たとえば、指定されたIDに一致するSummonerがない）。
- 指定されたパラメーターと一致するリソースがない。

**415 Unsupported Media Type**このエラーは、リクエストのボディがサポートされていないフォーマットであるため、サーバーがリクエストの処理を拒否していることを示します。

一般的な理由

- Content-Typeヘッダーが適切に設定されていない。

**429 Rate Limit Exceeded**このエラーは、アプリケーションが、与えられた期間中に許可される割り当てられた API 呼び出しの最大数を使い果たしたことを示します。クライアントがRate Limit Exceeded応答を受信した場合、クライアントはこの応答を処理し、Retry-Afterヘッダーで示された期間（秒）の間、今後のAPIコールを停止する必要があります。このポリシーに違反したアプリケーションは、APIの整合性を維持するためにアクセスが無効になる場合があります。レート制限を受けたかどうかの判断と回避方法の詳細については、以下のレート制限に関するドキュメントを参照してください。

一般的な理由

- 規制されていないAPIコール。

### 5xx エラーコード {{ id: '5xx-error-code' }}

5xxクラスまたはエラーコードは、サーバーがエラーまたはリクエストの実行が不可能であることを認識していることを示します。以下は、APIを使用する際に遭遇する可能性のある、最も一般的な5xxクラスのエラーコードです。

**500 Internal Server Error**このエラーは、サーバーが API リクエストを処理できない予期せぬ状況または例外を示します。

**503 Service Unavailable**このエラーは、サーバーが原因不明のリクエストを処理するために、現在利用できないことを示します。Service Unavailableレスポンスは、一時的な状態を意味し、しばらくすると緩和されるでしょう。

## レート制限  {{ id: 'rate-limiting' }}

Riot Games APIの使用を制御するために、一定時間内にエンドポイントにアクセスできる回数に制限を設けています。この制限は、不正使用を最小限に抑え、高い安定性を維持し、APIを支える基盤システムを過負荷から保護するために設けられています。基礎となるシステムは、私たちのゲームを動かしているのと同じシステムなので、もし過負荷になれば、プレイヤーの体験が損なわれてしまいます。

### レートリミッタータイプ {{ id: 'rate-limiting-type' }}

APIインフラで使用される制限には、アプリケーション、メソッド、サービスレート制限の3種類があります。

#### Application Rate Limits

最初のタイプの制限は、APIキー単位で実施され、**アプリケーションレート制限**と呼ばれます。アプリケーションのレート制限は、地域ごとに実施されます。ある地域のRiot Games APIエンドポイントへのすべての呼び出しは、その地域のそのキーのアプリケーション速度制限に対してカウントされます。たとえば、静的データAPIへの呼び出しは、アプリケーションの制限速度に対してカウントされません。

#### Method Rate Limits

2つ目のタイプの制限は、与えられたAPIキーに対してエンドポイント（または「メソッド」）ごとに適用され、メソッドレート制限と呼ばれます。**メソッドレート制限**は、地域ごとに実施されます。ある地域のRiot Games APIエンドポイントに行われたすべての呼び出しは、その地域の与えられたメソッドとAPIキーのメソッドレート制限に対してカウントされます。

#### Service Rate Limits

3つ目の制限は、サービスごとに適用され、**サービスレート制限**と呼ばれます。サービスレート制限も地域ごとに実施されます。ある地域のあるRiot Games APIサービスのエンドポイントに対して行われたすべての呼び出しは、その地域のそのサービスのサービスレート制限に対してカウントされます。サービス料金の制限が適用される場合、どのエンドポイントが料金制限のあるサービスの一部であるかを含め、文書化されます。

メソッドのレート制限とサービスのレート制限を混同しないでください。メソッド・レート・リミットは、各アプリケーションに個別に適用されます。サービス料金の制限は、サービスに適用され、サービスを呼び出すすべてのアプリケーションで共有されます。

#### Other Limits

APIインフラストラクチャによって強制されるこれらの制限は、提供されるデータへの唯一のゲートウェイではありません。あるエンドポイントの基礎となるサービスの中には、APIインフラストラクチャとは別に、独自のレート制限を実装しているものもあるかもしれない。このような場合、429エラーレスポンスが得られるが、レスポンスに含まれる**X-Rate-Limit-Type**ヘッダーは存在しない。APIエッジインフラによってレート制限が実施された場合のみ、このヘッダーが含まれる。

私たちのポリシーとして、レート制限がどのように機能するかの詳細は明らかにしませんが、コードの目的のために、最初のAPIコールを行ったときにバケットが開始されると仮定することは可能です。

## バージョニング {{ id: 'versioning' }}

2019年現在、これらのバージョニングガイドラインは、すべてのRiot Games APIに適用されます。ただし、将来的に変更される可能性があります。製品を開発する各ゲームのドキュメントを確認し、最新の情報を入手するようにしてください。

現在、League of LegendsのAPIはすべてバージョン4です。APIパスにマイナーバージョンを含めることはなくなりました。

## デプロイメント  {{ id: 'deprecation' }}

2019年現在、これらの非推奨ガイドラインは、すべてのRiot Games APIに適用されます。ただし、将来的に変更される可能性があります。最新の情報を得るために、製品を開発している各ゲームのドキュメントを必ず確認してください。

APIを廃止、または変更する場合は常に、そのAPIの新旧両バージョンを60日間サポートすることを目標としています。その後、旧バージョンは非推奨となります。場合によっては、非推奨の期間を60日よりも長くしたり短くしたりする必要があります。非推奨期間については、ソーシャルメディアやDeveloper Relations<a target="_blank" href="https://www.riotgames.com/en/DevRel">ブログ</a>を通じて、開発者の皆様にお知らせします。

開発者は、Riot Games APIを使用した製品を開発する際に注意する必要があります。APIが非推奨となり更新された場合、以前のバージョンは使用できなくなります。有効なAPIキーから非推奨のエンドポイントに呼び出すと、エラーコードが発生します。
